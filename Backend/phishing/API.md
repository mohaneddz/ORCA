# Phishing Simulation Engine — API Reference

All endpoints live under `http://<host>/api/`.  
**Authentication**: every endpoint requires `Authorization: Token <key>` in the request header, except the click-tracking endpoint which is intentionally public.

---

## Templates

### `GET /api/phishing/templates/`
List all saved phishing templates.

**Response**
```json
{
  "templates": [
    {
      "id": "uuid",
      "attack_type": "IT_RESET",
      "language": "EN",
      "difficulty": 2,
      "subject": "Action Required: Your Microsoft 365 session will expire in 12 hours",
      "sender_name": "Acme Corp IT Helpdesk",
      "sender_domain": "microsoft365-helpdesk-portal.com",
      "created_at": "2026-05-02T10:00:00Z"
    }
  ]
}
```

---

### `POST /api/phishing/templates/`
Create a template manually.

**Body**
```json
{
  "attack_type": "IT_RESET",
  "language": "EN",
  "difficulty": 2,
  "subject": "Your password expires today",
  "body": "Click here to reset: {{tracking_url}}",
  "sender_name": "IT Support",
  "sender_domain": "it-helpdesk.net"
}
```

| Field | Values |
|---|---|
| `attack_type` | `IT_RESET` · `INVOICE` · `DELIVERY` · `HR_UPDATE` |
| `language` | `EN` · `FR` · `AR_MSA` · `AR_DARIJA` |
| `difficulty` | `1` (obvious) · `2` (plausible) · `3` (spear-phishing) |

Use `{{tracking_url}}`, `{{org_name}}`, `{{employee_name}}` as placeholders in subject/body/sender_name.

**Response `201`**
```json
{ "id": "uuid" }
```

---

### `POST /api/phishing/templates/generate/` 🤖
AI-generate or library-generate a template in seconds.

**Body**
```json
{
  "attack_type": "INVOICE",
  "language": "FR",
  "difficulty": 2,
  "employee_id": "uuid",
  "save": true
}
```

| Field | Required | Notes |
|---|---|---|
| `attack_type` | ✅ | |
| `language` | ✅ | |
| `difficulty` | ❌ | Auto-recommended from employee history if omitted |
| `employee_id` | ❌ | Personalises body with employee name (needed for difficulty 3) |
| `save` | ❌ | `true` persists as a `PhishingTemplate` record |

> **AI behaviour**: if `NVIDIA_API_KEY` is set in the environment the template body is generated by `meta/llama-4-maverick-17b-128e-instruct` via the NVIDIA NIM API. If the key is absent or the call fails, the engine falls back to the built-in culturally-localised static library (48 templates: 4 attack types × 4 languages × 3 difficulties).

**Response `200`** (or `201` if `save: true`)
```json
{
  "id": "uuid",
  "attack_type": "INVOICE",
  "language": "FR",
  "difficulty": 2,
  "subject": "Facture #FAC-2026-0441 en attente de votre validation",
  "body": "Bonjour,\n\n...",
  "sender_name": "Acme Corp Comptabilité Fournisseurs",
  "sender_domain": "accounts-payable-invoicing.net"
}
```

## Campaigns

### `GET /api/phishing/campaigns/`
List all campaigns for the organisation.

### `POST /api/phishing/campaigns/`
Create a new campaign (status: `DRAFT`).

**Body**
```json
{ "name": "Q2 Awareness Wave", "template_id": "uuid" }
```

---

### `POST /api/phishing/campaigns/<id>/launch/`
Launch campaign → creates simulation targets → **sends emails immediately**.

**Body** (optional — omit to target all active employees)
```json
{ "employee_ids": ["uuid1", "uuid2"] }
```

**Response**
```json
{
  "targets_created": 12,
  "status": "ACTIVE",
  "emails_sent": 12,
  "emails_failed": 0,
  "email_errors": []
}
```

> Emails are sent synchronously in the same request. Each email is an **HTML email** (with a styled blue CTA button) plus a plain-text fallback. The `From` display name is spoofed to match `sender_name` in the template.

---

### `POST /api/phishing/campaigns/<id>/complete/`
Mark a campaign as `COMPLETED`.

---

### `GET /api/phishing/campaigns/<id>/targets/`
List all simulation targets for a campaign (employee + sent/clicked timestamps).

## Click Tracking

### `GET /api/phishing/click/<token>/` — **Public (no auth)**
Called when an employee clicks the link in their simulation email.

1. Records `clicked_at` on the `PhishingSimulationTarget`
2. Auto-enrolls the employee in the training module matching the attack type + language
3. Returns an HTML awareness page explaining they were in a simulation

## Per-employee

### `GET /api/phishing/employees/<id>/difficulty/` 🤖
Get an auto-recommended simulation difficulty for an employee based on their recent click history.

**Response**
```json
{
  "employee_id": "uuid",
  "employee_email": "alice@example.com",
  "recommended_difficulty": 2,
  "history_window": 5,
  "failed_count": 1,
  "click_rate": 20.0
}
```

**Recommendation logic (last 5 simulations):**

| Click rate | Action |
|---|---|
| < 25% | Increase difficulty by 1 (max 3) |
| 25–59% | Keep current difficulty |
| ≥ 60% | Decrease difficulty by 1 (min 1) |
| < 2 simulations | Start at difficulty 1 |

---

### `POST /api/phishing/targets/<id>/sent/`
Manually mark a simulation target as sent (use this if you're sending emails externally and want to record it).

| Endpoint | AI involvement |
|---|---|
| `POST /api/phishing/templates/generate/` | Full template generation via NVIDIA NIM (`meta/llama-4-maverick-17b-128e-instruct`) when `NVIDIA_API_KEY` is set — falls back to static library |
| `GET /api/phishing/employees/<id>/difficulty/` | Rule-based adaptive difficulty that analyses click history and auto-adjusts the next simulation level |

## Difficulty guide

| Level | Description | Signals to look for |
|---|---|---|
| **1 — Easy** | Obvious phishing: misspellings, ALL-CAPS urgency, suspicious domain | Training for first-time awareness |
| **2 — Medium** | Plausible email: professional tone, believable domain (e.g. `microsoft365-helpdesk-portal.com`), realistic reference numbers | For employees with some baseline awareness |
| **3 — Hard** | Spear-phishing: employee name in subject, spoofed org domain (`it.acme-corp.com`), no obvious red flags | Advanced simulation for security-conscious employees |

## Attack type reference

| Type | Scenario | Psychological trigger |
|---|---|---|
| `IT_RESET` | Microsoft 365 / Azure AD credential expiry | Fear of losing system access |
| `INVOICE` | Vendor invoice approval, often forwarded from "a colleague" | Authority + time pressure |
| `DELIVERY` | DHL parcel held, customs duty required | Curiosity + small financial urgency |
| `HR_UPDATE` | Benefits enrollment deadline or policy acknowledgement | Fear of missing out on benefits |
